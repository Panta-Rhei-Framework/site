## Story in one sentence

Panta Rhei can be covered as a public research observatory: a website and GitHub organization arranged so that high-scope claims can be traced through obligations, construction, result status, verification routes, artifacts, and correction paths before anyone is asked for belief.

## What is new

Press Package 3 publishes the white paper [Building a Public Research Observatory for High-Scope Open Research]({{ '/publications/white-papers/building-a-public-research-observatory/' | relative_url }}). It is a technical blueprint for how `panta-rhei.site` implements inspectability as information architecture.

This is not a results package. It does not say the theory is proven, accepted, or externally validated. It explains how the public site and GitHub organization make the program easier to inspect.

The observatory is a review interface, not a proof of the theory.

## What can be said safely

- Panta Rhei publishes its public research system as an observatory, not only as papers or claims.
- The public route chain is Program -> Agenda -> Corpus -> Results -> Verify.
- Discover, Impact, Engage, and Publications support that chain with orientation, conditional relevance, open scrutiny, and citable artifacts.
- GitHub provides public source, issue, discussion, and contribution substrate.
- The architecture is a reusable inspection pattern for high-scope open research.

## What should not be said

- Do not say that the observatory proves or validates the scientific theory.
- Do not say that Jekyll, Pagefind, GitHub, Zenodo, or any infrastructure tool endorses Panta Rhei.
- Do not say that public visibility equals peer review or external acceptance.
- Do not imply that a review path replaces expert domain review, formal proof, bridge evaluation, or empirical testing.

## Suggested headline angle

The safest headline is not “Panta Rhei proves a theory of reality.” The safe Package 3 angle is:

> How should a public research program build the interface that lets outsiders inspect ambitious claims?

## One-sentence boundary

Inspection architecture is not validation; it is the public structure that makes validation, challenge, correction, review, and refusal easier to begin.
